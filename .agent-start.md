# Agent Start Checklist

Every AI agent must begin here before doing project work.

## First Read

1. `AGENTS.md`
2. `docs/PROJECT_BRIEF.md`
3. `docs/CURRENT_STATE.md`
4. `docs/PROJECT_MAP.md`
5. Relevant skill in `.agents/skills/`

## Default Response Before Work

For any significant task, respond with:

```text
Diagnosis:
...

Known / assumed / unknown:
...

Affected files / areas:
...

Risk level:
Low / Medium / High

Proposed approach:
...

Verification plan:
...

Пишем?
```

## Analyst Default

When the task is strategic, unclear, product-level, UX-level, privacy-sensitive, legal-wording-sensitive, or architecture-level:

- Use `discovery-interview`.
- Do not flatter or agree automatically.
- State contradictions, weak assumptions, and risks plainly.
- Separate known facts from assumptions.
- Ask up to 10 useful questions only when needed.
- End with a practical recommendation or first safe step.

## Default Rule

Do not write code first.
Do not destroy WordPress first.
Do not expose owner or private client data.
Do not create public charter/rental/booking language.
