# 3D Guide Preparation

## Purpose

Prepare the website architecture and capture checklist for a future interactive 3D yacht guide.

This skill is mandatory whenever presentation UX, gallery, media zones, deck navigation, walkthrough structure, hotspot language, or CSS placeholders may affect future 3D integration.

## Do Not

Do not build the 3D model before source material and structure are ready.

Do not force heavy 3D into the first mobile view.

Do not let the first controlled presentation block future 3D by using an unstructured gallery or unrelated marketing sections.

## Controlled Presentation Integration

Before full 3D exists, the controlled presentation should still prepare:

- stable yacht zone names
- zone-based media grouping
- lightweight deck/zone navigation
- hotspot-ready labels
- full-screen media viewer compatible with future 3D entry
- photo/video fallback for every future 3D zone
- optional "Guide" or "Explore" entry point only if it feels premium and not gimmicky

## Required 3D Zones

Map the yacht by:

- exterior profile
- swim platform
- cockpit
- flybridge
- bow area
- main deck / saloon
- galley
- helm
- master cabin
- VIP cabin
- guest cabins
- bathrooms
- crew/service areas if allowed
- engine room only if approved

## Capture Checklist

For every zone, capture:

- wide establishing shot
- slow left-to-right pan
- slow right-to-left pan
- ceiling/floor reference
- entry/exit transition
- detail shots
- vertical mobile clip
- horizontal cinematic clip
- still photos from all corners
- key materials/textures
- lighting on/off if relevant

## Output

1. Zone map
2. Required source media
3. Missing capture list
4. Hotspot plan
5. 3D model requirements
6. Website placeholder architecture
7. Mobile fallback plan
8. CSS/UX constraints for future integration
