# Discovery Interview

## Purpose

Convert an unclear request into a development-ready brief.

## Rules

- Do not code first.
- Do not over-interview if context is already sufficient.
- Extract the real goal, audience, roles, constraints, risks, and first safe step.
- Always consider premium positioning, mobile/tablet UX, privacy, legal wording, and migration risk.

## Output

1. Goal
2. Audience
3. Current state
4. Desired state
5. Core user flow
6. Screens/modules involved
7. Data/entities involved
8. Privacy/security concerns
9. Legal wording concerns
10. UX risks
11. First safe implementation step
