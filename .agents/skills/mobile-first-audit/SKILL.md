# Mobile First Audit

## Required Checks

Always check:

- iPhone viewport
- iPad/tablet viewport
- Safari toolbar behavior
- safe-area insets
- touch targets
- modal scroll
- video autoplay behavior
- image cropping
- page weight
- animation performance
- readable typography
- no horizontal overflow
- no tiny buttons
- no desktop-only hover dependency
- first screen feels complete on mobile
- gallery/media interaction works without long landing-page scrolling
- CSS system stays consistent across sections
- future 3D guide entry or placeholders do not overload first mobile view
- zone/deck navigation remains understandable without full 3D

## Premium Requirement

Mobile must feel intentional, not like a squeezed desktop page.

## Output

1. Mobile risks
2. Tablet risks
3. Performance risks
4. Media/crop risks
5. Touch/navigation risks
6. CSS consistency risks
7. Future 3D/mobile readiness risks
8. Required fixes
