# Handoff MD

## Purpose

Create clear markdown documents for developers and future AI agents.

## Required Sections

1. Project context
2. Goal
3. Current state
4. Desired result
5. Scope
6. Out of scope
7. Architecture notes
8. UX notes
9. Security/privacy notes
10. Files/modules involved
11. Risks
12. Acceptance criteria
13. First safe task

## Style

- direct
- structured
- no marketing fluff
- no vague “make it better”
- include constraints and protected behavior
